В данном задании требовалось сформировать схему БД и представить её в виде даталогической схемы данных.

В пункте 1 не требовалось продумывать бизнес логику, поэтому структура приведена как есть, без индексов, партиций и других средств, оптимизирующих запросы

Поскольку задание выполнялось в среде PostgreSQL, решение вопроса создания дерева категорий был выполнен через расширение ltree, хотя схожие инструменты есть и в других реляционных СУБД (например, hierarchyid в SQL Server).
В альтернативном случае лучше использовать рекурсивную таблицу (FK ссылается не на другую таблицу, а на себя; корневой элемент либо имеет NULL, либо ссылается на себя).

path для краткости формируется из id категорий и подкатегорий, а название категории выводится из атрибута name для соответствующего id.

Для добавления нескольких различных товаров была добавлена промежуточная таблица Ordered_goods, реализующая связь "многие-ко-многим" между Goods и Orders.
